# 石墨晶舟量測數據顯示與分析系統 規格書 v2

## 1. 專案名稱

**石墨晶舟量測數據顯示與分析系統**

---

## 2. 目前版本目標

目前先做一個可實際操作的第一階段版本，重點放在：

1. **手動輸入量測資料**
2. **資料寫入資料庫**
3. **可建立與管理 baseline**
4. **可在 Dashboard 依條件查詢與分析**
5. **先不開啟檔案上傳功能**
6. **先不處理儀器檔案 parser**
7. **畫面以工程分析與可讀性優先**

這一版不是展示用 mock-only 版本，而是要開始往可存資料、可查資料、可分析資料的方向走。

---

## 3. 目前確認的調整事項

### 3.1 量測資料建立方式
目前 **先只做手動輸入功能**，檔案上傳先不開。

### 3.2 檔案上傳
- UI 可保留「未開放 / 開發中」提示
- 不提供實際上傳功能
- 不做 CSV parser
- 不做檔案預覽
- 不做匯入 API

### 3.3 製程條件
原本使用下拉選單，現在改為 **手動輸入文字欄位**。  
原因是實驗條件不一定固定，手動輸入比較彈性，例如：

- 未製程前
- 常溫製程後
- 300°C 製程後
- 清洗後
- 鍍膜 10 次後
- 特定調整後

### 3.4 樣品編號
目前 **先拿掉樣品編號欄位**。  
第一版先專注在量測資料本身，之後如果需要再補回來。

### 3.5 欄位命名調整
量測表格欄位改為：

- **筆數**（原本 trial）
- **FREQ(Hz)**
- **LEVEL**
- **Rp**
- **Cp**
- **Rs**
- **Cs**

## 5. 第一版資料流程

### 手動輸入流程
使用者在「新增量測資料」頁面輸入：

- 資料名稱
- 製程條件
- 備註
- 是否指定 baseline
- 多筆量測資料

送出後流程：

1. 建立一筆 dataset 主資料
2. 建立多筆 measurement records
3. 若有綁定 baseline，一起紀錄 baseline 關聯
4. Dashboard 從資料庫讀取資料顯示

## 7. Prisma Schema 建議

```prisma
model BaselineProfile {
  id             String   @id @default(cuid())
  name           String
  conditionLabel String?
  freqHz         Float?
  level          Float?
  rp             Float?
  cp             Float?
  rs             Float?
  cs             Float?
  note           String?
  createdAt      DateTime @default(now())
  updatedAt      DateTime @updatedAt

  datasets       MeasurementDataset[]
}

model MeasurementDataset {
  id             String   @id @default(cuid())
  datasetName    String
  conditionLabel String
  note           String?
  baselineId     String?
  createdAt      DateTime @default(now())
  updatedAt      DateTime @updatedAt

  baseline       BaselineProfile? @relation(fields: [baselineId], references: [id])
  records        MeasurementRecord[]
}

model MeasurementRecord {
  id         String   @id @default(cuid())
  datasetId  String
  indexNo    Int
  freqHz     Float
  level      Float
  rp         Float
  cp         Float
  rs         Float
  cs         Float
  createdAt  DateTime @default(now())

  dataset    MeasurementDataset @relation(fields: [datasetId], references: [id], onDelete: Cascade)
}
```

## 8. 頁面規劃

## 8.1 `/measurements/new` — 新增量測資料頁

### 目標
先完成手動輸入量測資料並寫入資料庫。

### 區塊 A：基本資訊
欄位：

- 資料名稱
- 製程條件（手動輸入）
- 備註
- baseline 選擇

### baseline 選擇方式
建議改成：

- 不指定 baseline
- 選擇既有 baseline

不要只用 checkbox「設為 baseline」。  
因為 baseline 應該獨立管理。

### 區塊 B：量測資料表格
欄位：

- 筆數
- FREQ(Hz)
- LEVEL
- Rp
- Cp
- Rs
- Cs

操作：

- 新增一列
- 刪除一列
- 計算平均
- 儲存

### 區塊 C：檔案上傳
先不開啟，只顯示提示：

- 檔案匯入功能開發中
- 目前請先使用手動輸入

## 9. KPI 區重構

KPI 改成兩排。

### 第一排
1. **量測組數**
2. **Baseline 狀態**
3. **Baseline 最大偏差**
4. **Baseline 名稱 / 未設定**

### 第二排
1. **平均 Rp**
2. **平均 Cp**
3. **平均 Rs**
4. **平均 Cs**

---

## 10. KPI 顏色建議

- 量測組數：淡藍
- Baseline 狀態：
  - 正常：淡綠
  - 注意：淡黃
  - 異常：淡紅
  - 未設定：灰
- Baseline 最大偏差：淡紅 / 珊瑚
- Baseline 名稱：淡藍灰
- 平均 Rp：淡橘
- 平均 Cp：淡綠
- 平均 Rs：淡紫
- 平均 Cs：淡青藍

## 11. Dashboard 圖表規劃

## 11.1 圖表一：同條件量測趨勢圖

### 目的
看同一條件下多筆量測資料的變化。

### 建議做法
因為 R 與 C 數值量級不同，建議拆成兩張：

#### A. 電阻類趨勢圖
- 資料：Rp、Rs
- X 軸：筆數
- Y 軸：數值

#### B. 電容類趨勢圖
- 資料：Cp、Cs
- X 軸：筆數
- Y 軸：數值

### 控制項
每張圖自己的控制面板：

- 顯示平均線
- 顯示 baseline 線
- 顯示單筆量測

---

## 11.2 圖表二：Baseline 對照分析

### 目的
讓使用者知道：
- baseline 是多少
- 各筆量測是多少
- 平均是多少
- 偏差是多少

### 顯示方式
分兩部分：

#### A. baseline 對照表
欄位：

- 參數
- baseline 值
- 各筆量測值
- 平均值
- 偏差 %

#### B. baseline 對照圖
可切換參數：

- Rp
- Cp
- Rs
- Cs

圖中顯示：

- baseline 線
- 各筆量測點 / 線
- 平均線

### 控制項
- 參數切換
- 顯示 baseline
- 顯示平均
- 顯示單筆量測
- 顯示偏差 %

---

## 11.3 圖表三：不同製程條件比較

### 目的
讓工程師快速看出不同製程條件下的參數變化。

### 預設模式
**以平均值比較**，不要把所有單筆量測直接丟進圖表。

### 顯示方式
群組柱狀圖：

- X 軸：製程條件
- Y 軸：平均值

### 控制項
- 參數切換（Rp / Cp / Rs / Cs）
- 比較模式（平均值 / 中位數）
- 是否顯示 baseline 參考線

---

## 12. 查詢列與圖表控制分離原則

### 查詢列只負責：
- 資料篩選
- 選定分析範圍

### 圖表控制只負責：
- 控制該張圖怎麼顯示

不要把：
- 顯示平均值
- 顯示單筆量測
- 顯示 baseline
- 顯示偏差 %

這些放在最上面的查詢列。
